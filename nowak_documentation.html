Technical Architecture

The system uses Google Sheets as its database backend with Google Apps Script providing the API layer. The frontend is pure HTML/CSS/JavaScript hosted on GitHub Pages. All data syncs in real-time.

Component Technology Purpose
Database Google Sheets 8 tables storing all business data
Backend API Google Apps Script CRUD operations, business logic
Frontend HTML/CSS/JavaScript User interface, responsive design
Hosting GitHub Pages Free, fast, reliable hosting

üè¢ Suppliers

Suppliers are the companies you purchase products from. Each supplier has contact information and links to their items.

Adding a New Supplier

Click "+ Add New Supplier"

Find the green button at the top of the Suppliers tab

Fill Required Fields

Supplier Name is required. Add contact name, phone, email, and address as available.

Save

Archiving vs Deleting

Suppliers with existing orders cannot be deleted to preserve historical data. Instead, archive them to hide from active lists while keeping records intact.

üì¶ Items & Products

Items are the products you buy and sell. Each item is linked to a supplier and has pricing, inventory levels, and reorder points.

Stock Status Indicators

Color Status Meaning
üü¢ Green In Stock Quantity above reorder point
üü° Yellow Low Stock Below reorder point but > 0
üî¥ Red Out of Stock Zero quantity available

Setting Reorder Points

Reorder points trigger low-stock warnings. Set them based on typical sales velocity and lead time from suppliers. The Reorder Report shows all items below their reorder point.

üõí Purchase Orders

üìä Inventory

The Inventory tab shows real-time stock levels for all items with filtering and manual adjustment capabilities.

Automatic vs Manual Updates

  • Automatic: Receiving POs increases stock; delivering SOs decreases stock
  • Manual: Use adjustments for physical counts, damage, shrinkage, or corrections

Making Adjustments

When making manual adjustments, always select an appropriate reason:

  • Physical Count - Correcting after counting
  • Damaged Goods - Writing off damaged product
  • Shrinkage - Unexplained loss
  • Found Stock - Discovered extra inventory
  • Returned Item - Customer returns

‚ùì FAQ

Can I access this from multiple devices?

Yes! The system works on any device with a web browser. Your data syncs in real-time across all devices.

Is my data backed up?

Yes, Google Sheets automatically saves and maintains version history. You can access previous versions through Google Drive.

Can multiple people use the system at once?

Yes, multiple users can work simultaneously. Changes sync in real-time.

What browsers are supported?

Chrome, Firefox, Safari, and Edge all work well. For best experience, use the latest version of your browser.

How do I reset test data?

Contact your administrator to reset the test environment data. This does not affect production.
